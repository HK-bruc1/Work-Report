# 系统时钟源选择

> 本文档分析 AC701N TWS 耳机公版 SDK（`701公版SDK/SDK/`）中所有与时钟相关的设计。

---

## 一、时钟源分类

AC701N 的时钟源分为两大类：**外部晶振**和**片内独立 RC 振荡器**，两者相互独立，没有从属关系。

```
┌─────────────────────────────────────────────────────────────────┐
│                    片内独立 RC 振荡器                             │
│  （靠片内 RC 时间常数自激振荡，完全不依赖外部器件，精度低有温漂）    │
│                                                                  │
│   LRC   ≈ 200 kHz  ──► 低功耗睡眠时钟 / 可选作 RTC 时钟源        │
│   HRC   ≈  16 MHz  ──► P11 子系统备用时钟 / GPTIMER 时钟源       │
│   RC    ≈ 250 kHz  ──► P11 子系统极低功耗待机                    │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                    外部晶振（板级器件）                            │
│  （石英晶振接芯片引脚，精度高，频率稳定）                           │
│                                                                  │
│  XOSC 24MHz（主晶振，接 XOSC 引脚）                               │
│      │                                                           │
│      ├──► 片内 PLL 倍频 ──► CPU 主时钟（24 ~ 240 MHz）            │
│      │                                                           │
│      └──► BT OSC 24/48 MHz ──► STD 12M / 24M / 48M（各外设）     │
│                                                                  │
│  RTC OSC 32.768 kHz（独立小晶振，接 RTC_OSC 引脚，可选）           │
│      └──► 片上 RTC 外设（计时寄存器）                              │
│           ※ 若不接此晶振，RTC 可改用 LRC 或 BT OSC 分频代替        │
└─────────────────────────────────────────────────────────────────┘
```

## 二、各时钟源详解

### 2.1 片内 RC 振荡器（三种）

RC 振荡器的工作原理是靠片内电阻（R）和电容（C）构成充放电回路自激振荡，频率由片内 RC 时间常数决定。**它们与外部晶振没有任何关系，互相独立。**

#### LRC（Low RC，约 200 kHz）

- **频率**：约 200 kHz，`p11_app.h` 中定义 `LRC_Hz_DEFAULT = (200 * 1000L)`
- **精度**：低，有温漂，仅适合低功耗粗计时，不适合精确走时
- **主要用途**：
  - 系统进入 Sleep/DeepSleep 后作为 CPU 时钟（PLL 关闭以省电）
  - P11 子系统低功耗时钟：`P11_SYS_CLK_LRC_OSC`
  - 可选作 RTC 计时时钟源：`CLK_SEL_LRC`
  - 可选作 GPTIMER 时钟源：`GPTIMER_CLK_SRC_LRC`
  - 可选作 VAD 低功耗时钟：`VAD_CLOCK_USE_LRC`
- **公版低功耗配置**（`sdk_config.h`）：

  ```c
  #define TCFG_LOWPOWER_OSC_TYPE    OSC_TYPE_LRC
  ```

- **寄存器初始化**（`interface/driver/cpu/br28/asm/power/p11_app.h`）：

  ```c
  #define LRC_CON0_INIT   (1<<5) | (1<<0)   // RC32K_RPPS_S1 + RC32K_EN
  #define LRC_CON1_INIT   (1<<6) | (1<<0)   // RC32K_CAP_S2 + RC32K_RNPS_S0
  ```

#### HRC（High RC，约 16 MHz）

- **频率**：约 16 MHz
- **精度**：低，有温漂
- **主要用途**：
  - P11 子系统备用高频时钟：`P11_SYS_CLK_RC16M`
  - 可选作 PLL 参考源（无外部晶振时的备用方案）：`PLL_REF_HRC`
  - 可选作 GPTIMER 时钟源：`GPTIMER_CLK_SRC_RC16M`
- **定义位置**：`interface/driver/cpu/periph/clock.h`（`enum pll_ref_source`）

#### RC 250 kHz

- **频率**：约 250 kHz
- **精度**：低，有温漂
- **主要用途**：P11 子系统最低功耗待机状态 `P11_SYS_CLK_RC250K`
- **定义位置**：`interface/driver/cpu/br28/asm/power/p11_app.h`

---

### 2.2 外部主晶振 XOSC（24 MHz）

- **物理形态**：板级外接石英晶振，接芯片 XOSC 引脚
- **频率**：24 MHz，由 `TCFG_CLOCK_OSC_HZ = 24000000` 配置
- **两种接法**：
  - 单端：`PLL_REF_XOSC`
  - 差分：`PLL_REF_XOSC_DIFF`（公版默认）
- **直接派生出**：PLL 时钟（倍频）、BT OSC（直通或倍频）
- **定义位置**：`interface/driver/cpu/periph/clock.h`（`enum pll_ref_source`）
- **初始化位置**：`cpu/br28/setup.c:228`

  ```c
  clk_early_init(PLL_REF_XOSC_DIFF, TCFG_CLOCK_OSC_HZ, 240 * MHz);
  ```

---

### 2.3 PLL（片内锁相环，CPU 主时钟）

- **来源**：以 XOSC 24 MHz 为参考，片内 PLL 倍频输出
- **最高频率**：
  - 普通耳机模式：192 MHz
  - 耳机盒模式（`CONFIG_EARPHONE_CASE_ENABLE`）：240 MHz
- **支持的频率档位**（`apps/common/clock_manager/clock_manager.c:46`）：

  ```
  24, 48, 60, 64, 76, 80, 96, 120, 128, 140,
  160, 200, 240, 280, 320  （单位 MHz）
  ```

- **配置入口**（`board/br28/board_ac701n_demo_cfg.h`）：

  ```
  TCFG_CLOCK_SYS_SRC  = SYS_CLOCK_INPUT_PLL_BT_OSC
  TCFG_CLOCK_SYS_HZ   = 24000000     // 系统初始频率
  TCFG_CLOCK_MODE     = CLOCK_MODE_ADAPTIVE
  ```

---

### 2.4 BT OSC / STD 标准时钟

- **来源**：XOSC 24 MHz 经过芯片内部 BT 模块处理，输出 24 MHz 或 48 MHz
- **派生出各路标准时钟**：STD 12M、STD 24M、STD 48M，供各外设使用
- **GPTIMER5 系统定时器**默认使用 STD 12M，见第五节

---

### 2.5 RTC OSC（外部独立小晶振，32.768 kHz）

- **物理形态**：板级外接 32.768 kHz 石英晶振，接芯片 RTC_OSC 引脚，**独立于主晶振**
- **用途**：为片上 RTC 外设提供高精度计时心跳，掉电后只要 RTC 域有供电就持续走时
- **是否必须**：**可选**。不接时可改用 LRC 或 BT OSC 分频，但精度下降
- **公版默认**（`apps/earphone/include/app_config.h:542`）：

  ```c
  #define RTC_CLK_RES_SEL    CLK_SEL_32K
  ```

---

### 2.6 XCLK（外部时钟输入）

- **物理形态**：外部信号源从芯片引脚输入
- **用途**：作为 PLL 参考 `PLL_REF_XCLK`，仅用于测试或特殊场景

## 三、RTC 片上外设

RTC 是芯片内部的一个外设（计时寄存器 + 闹钟逻辑），本身不产生时钟，需要选择一个外部时钟源驱动它计数。

### 3.0 RTC 电源域与运行行为

RTC 外设位于芯片内部的 **P33 RTCVDD 电源域**，该域直接由电池（VBAT）供电，独立于主系统电源。这与 STM32 的 VBAT 引脚接备用纽扣电池的设计原理相同，区别在于 TWS 耳机没有独立备用电池，直接复用耳机主电池作为 VBAT 来源。

**各场景下 HW_RTC 的运行行为：**

| 场景 | P33 RTCVDD 供电 | RTC 是否继续走时 |
|---|---|---|
| 正常运行 | 有 | 是 ✓ |
| Light Sleep | 有 | 是 ✓ |
| Sleep / Deep Sleep | 有 | 是 ✓（时钟源切换到 32K 晶振或 LRC，功耗极低）|
| Soft-Off（软关机） | 有 | 是 ✓（P33 RTCVDD 始终保持，RTC 计数器不停）|
| 硬断电（电池取出/耗尽） | 无 | 否 ✗（时间丢失，重新上电后需重新设置时间）|

> 只要耳机电池有电，不论系统处于任何状态（包括软关机），RTC 就一直在走时。这正是 RTC 区别于 jiffies 的根本价值：jiffies 是系统运行时间计数器，Soft-Off 后归零；RTC 是真实挂钟时间，Soft-Off 后仍然连续。

**芯片电源域层次（低功耗相关）：**

```
电池 VBAT
    │
    ├──► P33 RTCVDD域（始终供电）
    │        ├─ RTC 计时器 / 闹钟
    │        ├─ 32K 晶振驱动电路
    │        └─ 唤醒控制逻辑
    │
    ├──► P11 低功耗子系统（Soft-Off 时有限供电）
    │        ├─ P11 RAM（掉电保持，用于存 RTC 时间备份）
    │        └─ 唤醒消息通路
    │
    └──► 主系统（Soft-Off 时断电）
             ├─ CPU / PLL / GPTIMER5（jiffies 来源）
             └─ 各外设
```

**`poweroff_save_rtc_time()` 的作用：**

软关机前，`power_soff_callback()` 会调用此函数（`cpu/br28/power/power_app.c:107`）。虽然 HW_RTC 在 Soft-Off 时并不停止，但此函数额外将当前 RTC 时间和 LRC 校准数据写入 **P11 RAM**（`p11_sys_time` 结构体），作为双重保险：

```c
struct p11_sys_time {
    u32 mask;
    struct sys_time   ram_time;      // 当前时间备份
    struct _rtc_trim  ram_lrc_trim;  // LRC 频率校准数据
};
```

P11 RAM 在 Soft-Off 期间保持供电，系统重启后可用此备份校验 / 恢复时间，同时利用 `ram_lrc_trim` 补偿 LRC 在深度睡眠期间的频率漂移误差。

### 3.1 RTC 可选时钟源

**定义位置**：`interface/driver/device/rtc/rtc_dev.h`

| 枚举值 | 频率 | 来源 | 精度 | Soft-Off 期间继续走时 |
|---|---|---|---|---|
| `CLK_SEL_32K` (1) | 32.768 kHz | 外部独立 RTC 小晶振（P33 域） | 高 | 是 ✓ |
| `CLK_SEL_BTOSC_DIV1` (2) | 24 MHz | BT OSC 不分频（依赖 P11） | 高 | 否 ✗ |
| `CLK_SEL_BTOSC_DIV2` (3) | 12 MHz | BT OSC 2 分频（依赖 P11） | 高 | 否 ✗ |
| `CLK_SEL_LRC` (4) | ~200 kHz | P33 域内部 RC 振荡器 | 低 | 是 ✓ |

> **容易混淆的关键点：`CLK_SEL_LRC` 中的 LRC 不是 P11 系统 LRC。**
>
> 芯片中存在两个不同的 LRC：
>
> | | P11 系统 LRC | RTC 的 `CLK_SEL_LRC` |
> |---|---|---|
> | 所在域 | P11 主系统域 | **P33 RTCVDD 域** |
> | 供电来源 | 主系统电源 | VBAT（电池直接供电） |
> | Soft-Off 时状态 | P11 断电，**停止** | P33 始终有电，**继续运行** |
> | 主要用途 | CPU 低功耗睡眠时钟 | RTC 计时心跳（无外部 32K 晶振时的备选） |
>
> 因此，**板子上没有焊 32K 晶振时选 `CLK_SEL_LRC`，RTC 在 Soft-Off 期间照样能走时**，与 `CLK_SEL_32K` 的唯一差距是精度：石英晶振精度约 ±20ppm，P33 内部 RC 振荡器因温漂误差可达 ±1~3%（相当于每天偏差约 15 分钟到 45 分钟）。
>
> `CLK_SEL_BTOSC_DIV1/2` 依赖 P11 的 BT OSC 电路，Soft-Off 后 P11 断电，BT OSC 停振，RTC 也随之停止，**不能用于需要 Soft-Off 走时的场景**。

## 四、P11 子系统时钟选择

P11 是芯片内负责低功耗管理的子系统，有独立的时钟多路选择器。

**定义位置**：`interface/driver/cpu/br28/asm/power/p11_app.h`

```c
enum P11_SYS_CLK_TABLE {
    P11_SYS_CLK_RC250K = 0,     // 片内 RC 250 kHz（最低功耗）
    P11_SYS_CLK_RC16M,          // 片内 HRC 16 MHz
    P11_SYS_CLK_LRC_OSC,        // 片内 LRC ~200 kHz
    P11_SYS_CLK_BTOSC_24M,      // 外部主晶振经 BT OSC 24 MHz
    P11_SYS_CLK_BTOSC_48M,      // 外部主晶振经 BT OSC 48 MHz

#define P11_SYS_CLK_SEL(x)   (P11_CLOCK->CLK_CON0 = x)
```

```c
#define CLOCK_KEEP(en) \

        P11_CLOCK->CLK_CON1 |= BIT(14); \
```

## 五、系统定时器 / jiffies 时基

### 5.1 硬件定时器 GPTIMER5（系统时基）

**定义位置**：`interface/driver/cpu/br28/asm/gptimer_hw.h`

系统固定使用 TIMER5，时钟源为 STD 12M（来自 BT OSC），4 分频后得到 3 MHz 计数时钟。

```c
#define GPTIMER_SYS             GPTIMER5

#define GPTIMER_SYS_INIT()  do{ \
    GPTIMER_SYS->CON = GPTIMER_PND_CLR     \
                     | GPTIMER_CLK_SRC_STD12M \
                     | GPTIMER_CLK_DIV_4;  \
    GPTIMER_SYS->PRD = 0;                  \
    GPTIMER_SYS->CNT = 0;                  \
}while(0)
```

GPTIMER 可选时钟源（6 个 TIMER 通道均支持独立配置）：

| 宏 | 时钟来源 |
|---|---|
| `GPTIMER_CLK_SRC_LSB` | 低速总线时钟 |
| `GPTIMER_CLK_SRC_OSC` | OSC 时钟 |
| `GPTIMER_CLK_SRC_LRC` | 片内 LRC ~200 kHz |
| `GPTIMER_CLK_SRC_RC16M` | 片内 HRC 16 MHz |
| `GPTIMER_CLK_SRC_STD12M` | STD 12M（系统定时器默认） |
| `GPTIMER_CLK_SRC_STD24M` | STD 24M |
| `GPTIMER_CLK_SRC_STD48M` | STD 48M |
| `GPTIMER_CLK_SRC_IOMUX` | GPIO 复用输入 |

### 5.2 jiffies 时间基准

GPTIMER5 每 10 ms 触发一次中断，中断处理中 `jiffies++`，`jiffies_msec()` 返回 `jiffies × 10`，即**本次上电启动以来的累计运行毫秒数**。其时钟来源同样是外部主晶振经 BT OSC 派生的 STD 12M，**与 STM32 引入 OS 后 SysTick 作为系统调度时钟的作用完全相同**，区别仅在于 SysTick 是 ARM Cortex-M 内核自带的专用定时器，而此处用的是片上通用 GPTIMER5。

完整时钟链路：

```
XOSC 24MHz（外部主晶振）
    └──► BT OSC
         └──► STD 12MHz
              └──► GPTIMER5（÷4）──► 3MHz 计数时钟
                   └──► 每 30000 次计数触发一次中断（10ms）
                        └──► jiffies++
                             └──► jiffies_msec() = jiffies × 10
```

与 STM32 + OS 的对比：

| | STM32 + OS | AC701N |
|---|---|---|
| 时钟来源 | 外部晶振 → PLL → AHB 总线时钟 | XOSC → BT OSC → STD 12M |
| 硬件定时器 | SysTick（ARM 内核自带） | GPTIMER5（片上通用定时器） |
| 计数频率 | 通常 1MHz 或 AHB 频率 | 3MHz（STD 12M ÷ 4） |
| 中断周期 | 1ms（OS 常用配置） | 10ms（HZ = 100） |
| 累计计数变量 | `xTickCount`（FreeRTOS）等 | `jiffies` |
| 毫秒查询接口 | `xTaskGetTickCount() × tick周期` | `jiffies_msec()` |
| OS 用途 | 任务调度 tick | 杰理自研 OS 任务调度 tick |

**jiffies_msec() 在各场景下的行为：**

| 场景 | GPTIMER5 状态 | jiffies 行为 |
|---|---|---|
| 正常运行 | 持续计数（STD 12M 供给） | 正常递增 |
| Light Sleep（浅睡眠） | STD 12M 保持，继续计数 | 正常递增，**无断层** |
| Sleep / Deep Sleep | CPU 切换到 LRC，GPTIMER5 暂停 | 暂停；唤醒时通过 `resume(priv, usec)` 回调将睡眠期间经过的微秒数**补偿回 jiffies**，对上层透明 |
| Soft-Off（软关机） | 完全停止 | 下次从软关机唤醒后**从 0 重新开始** |
| 硬断电重新上电 | 完全停止 | 重新上电后**从 0 重新开始** |

> **核心结论**：`jiffies_msec()` 记录的是**本次上电周期内的累计运行时间**，不是挂钟时间（wall clock）。Sleep/DeepSleep 有补偿机制保证连续性，但 Soft-Off 和硬断电后归零。这正是系统需要单独维护 RTC 来记录真实时刻的原因。

补偿机制由电源管理框架负责（`interface/driver/power/power_manage.h`）：

```c
struct low_power_operation {
    u32 (*get_timeout)(void *priv);
    void (*suspend_post)(void *priv, u32 usec);
    void (*resume)(void *priv, u32 usec);      // usec：睡眠期间经过的微秒数
```

## 七、Clock Manager（时钟管理器）

**位置**：`apps/common/clock_manager/`

时钟管理器采用**申请-释放**机制，维护所有模块的时钟需求，取最大值设为系统频率。

### 7.1 公共接口

```c
int  clock_alloc(const char *name, u32 clk);   // 申请时钟频率
int  clock_free(const char *name);             // 释放时钟频率
int  clock_lock(const char *name, u32 clk);    // 锁定固定频率（不参与动态调整）
int  clock_unlock(char *name);                 // 解除锁定
void clock_refurbish(void);                    // 立即重新计算并应用系统频率
void clock_manager_dump(void);                 // 打印所有时钟申请节点
```

### 7.2 自适应调频算法

```c
#define CLOCK_DETECT_PERIOD    (4 * 1000)  // 检测周期：4 秒
#define CLOCK_DETECT_COUNTER   (7)         // 连续 7 次达标后频率稳定
#define CLOCK_MINIMUM_FREQ     (24 * MHz)  // 系统最低保证频率

// CPU 使用率门限（普通模式）
static u8 cpu_usage_limit[] = {50, 55, 60};
// CPU 使用率门限（耳机盒模式）
static u8 cpu_usage_limit[] = {75, 80, 85};
```

调频流程：
1. 模块调用 `clock_alloc("name", freq)` 登记需求
2. 触发 `clock_refurbish()`，系统频率 = 所有需求中的最大值
3. 每 4 秒读取一次 CPU 使用率（`os_cpu_usage()` + 音频引擎使用率）
4. 按三阶段算法上调或下调频率档位
5. 连续 7 次检测达标后停止检测，频率稳定

## 九、低功耗时钟切换

**定义位置**：`interface/driver/cpu/br28/asm/power/power_api.h`

```c
struct low_power_param {
    u8  osc_type;       // 低功耗时钟源类型
    u32 btosc_hz;       // BT OSC 频率
    u8  rtc_clk;        // RTC 时钟源
    u32 osc_delay_us;   // 晶振起振等待时间
    u32 t1, t2, t3_lrc, t4_lrc, t3_btosc, t4_btosc;  // 唤醒时序参数
};

enum {
    OSC_TYPE_LRC    = 0,  // 低功耗使用片内 LRC（最省电，精度低）
    OSC_TYPE_BT_OSC = 1,  // 低功耗使用 BT OSC（精度高，功耗略高）
};
```

公版配置（`sdk_config.h`）：`#define TCFG_LOWPOWER_OSC_TYPE  OSC_TYPE_LRC`

低功耗时钟切换路径：

```
正常运行：XOSC → PLL → CPU（24~240 MHz）
    ↓ 进入 Sleep / DeepSleep
    PLL 关闭，切换到片内 LRC ~200 kHz（或保持 BT OSC 维持蓝牙）
    ↓ RTC 闹钟 / GPIO / 定时中断唤醒
    LRC 继续运行 → 等待 XOSC 起振稳定 → PLL 重新锁定 → 恢复正常
    ↓ 从 RTC 外设恢复当前时间
    poweroff_save_rtc_time() / rtc_read_time() / rtc_write_time()
```

## 十一、各时钟源对比总结

| 时钟源 | 频率 | 精度 | 是否需外部器件 | 独立性 | 主要用途 |
|---|---|---|---|---|---|
| 片内 LRC | ~200 kHz | 低（温漂大） | 否 | 完全独立 | 低功耗睡眠时钟 |
| 片内 HRC | ~16 MHz | 低（温漂大） | 否 | 完全独立 | P11 备用，GPTIMER |
| 片内 RC | ~250 kHz | 低（温漂大） | 否 | 完全独立 | P11 极低功耗待机 |
| XOSC | 24 MHz | 高 | 是（主晶振） | 外部输入 | PLL 参考，BT OSC |
| PLL | 24~240 MHz | 高 | 依赖 XOSC | 片内倍频 | CPU 主时钟 |
| BT OSC / STD | 12/24/48 MHz | 高 | 依赖 XOSC | 片内处理 | 各外设标准时钟 |
| RTC OSC | 32.768 kHz | 高 | 是（独立小晶振，可选） | 外部输入 | RTC 精确走时 |
| XCLK | 外部输入 | — | 是（外部信号） | 外部输入 | 测试/特殊场景 |
