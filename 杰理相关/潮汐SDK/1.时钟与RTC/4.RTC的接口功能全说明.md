# RTC 接口功能全说明

> 源文件：`interface/driver/device/rtc/rtc_dev.h`、`interface/driver/cpu/br28/asm/power/rtc_app.h`
> 应用封装：`apps/common/include/rdx_rtc.h` / `apps/earphone/rdx_rtc.c`

---

## 一、数据结构

### 1.1 `struct sys_time`（`interface/utils/sys_time.h`）

```c
struct sys_time {
    u16 year;   // 年，如 2025
    u8  month;  // 月，1–12
    u8  day;    // 日，1–31
    u8  hour;   // 时，0–23
    u8  min;    // 分，0–59
    u8  sec;    // 秒，0–59
};
```

所有 RTC 读写接口的时间载体，也是闹钟时间的载体。

---

### 1.2 `struct rtc_dev_platform_data`

RTC 设备注册时使用的平台配置结构体，在 `device_config.c` 中通过宏填写并注册。

| 字段 | 类型 | 说明 |
|---|---|---|
| `default_sys_time` | `const struct sys_time *` | 首次上电（或 RTCVDD 断电后）写入的默认时间 |
| `default_alarm` | `const struct sys_time *` | 首次上电写入的默认闹钟时刻 |
| `rtc_clk` | `enum RTC_CLK` | RTC 时钟源，见 §2.1 |
| `rtc_sel` | `enum RTC_SEL` | 使用硬件 RTC 还是软件虚拟 RTC，见 §2.2 |
| `x32xs` | `u8` | 32K 晶振内部负载电容档位（通常保持 0，由宏末尾自动填充） |
| `cbfun` | `void (*)(u8)` | 闹钟触发时的回调函数，`NULL` 表示不使用 |

**使用宏**

```c
RTC_DEV_PLATFORM_DATA_BEGIN(rtc_data)
    .default_sys_time = &def_sys_time,
    .default_alarm    = &def_alarm,
    .rtc_clk          = RTC_CLK_RES_SEL,
    .rtc_sel          = HW_RTC,
    .cbfun            = my_alarm_callback,
RTC_DEV_PLATFORM_DATA_END()
```

`RTC_DEV_PLATFORM_DATA_END()` 宏会自动填充 `.x32xs = 0` 并关闭大括号。

---

### 1.3 `struct _rtc_trim`

LRC（片内低速 RC 振荡器）校准参数，供虚拟 RTC 精度补偿使用，一般不需要直接操作。

| 字段 | 类型 | 说明 |
|---|---|---|
| `sfr_time_to_sec` | `u64` | 上次写入 SFR 时对应的秒数基准 |
| `now_time_to_sec` | `u64` | 当前时间的秒数基准 |
| `last_time_remain` | `u64` | 上次计算剩余的秒数偏差 |

---

### 1.4 `struct p11_sys_time`

P11 掉电域（VDDIO 断电依旧保存的 RAM）中存储的时间快照，用于 LRC 虚拟 RTC 在掉电后恢复时间。

| 字段 | 类型 | 说明 |
|---|---|---|
| `mask` | `u32` | 有效性标志，用于判断 P11 数据是否可信 |
| `ram_time` | `struct sys_time` | 存储的时间快照 |
| `ram_lrc_trim` | `struct _rtc_trim` | 存储的 LRC 校准参数 |

---

### 1.5 `struct rtc_dev_fun_cfg`

RTC 驱动函数表，用于硬件 RTC / 虚拟 RTC 的统一抽象，由底层驱动填充，上层通过 `rtc_config_init` 绑定后即可通过 `rtc_read_time` 等统一接口调用，不需要直接操作此结构体。

---

## 二、枚举

### 2.1 `enum RTC_CLK`——时钟源

| 枚举值 | 数值 | 说明 |
|---|---|---|
| `CLK_SEL_32K` | 1 | 外部 32.768 kHz 晶振（最高精度，需要板上焊接晶振接 RTC_OSC 引脚） |
| `CLK_SEL_BTOSC_DIV1` / `CLK_SEL_12M` | 2 | BT OSC 12 MHz 分频，精度与蓝牙晶振相关 |
| `CLK_SEL_BTOSC_DIV2` / `CLK_SEL_24M` | 3 | BT OSC 24 MHz 分频 |
| `CLK_SEL_LRC` | 4 | 片内 LRC（约 32 kHz 低速 RC），无需外部晶振，温漂较大（±5 % 量级） |

**选择原则：**

- 板上有 32K 晶振 → 选 `CLK_SEL_32K`，精度最高。
- 板上没有 32K 晶振 → 必须选 `CLK_SEL_LRC`，否则 RTC 不走时。

---

### 2.2 `enum RTC_SEL`——RTC 实现方式

| 枚举值 | 说明 |
|---|---|
| `HW_RTC` | 硬件 RTC：P33 RTCVDD 域的实体计时电路，软关机后继续走时，精度最高 |
| `VIR_RTC` | 虚拟 RTC：用 LRC 计数 + P11 RAM 保存时间快照，精度低于硬件 RTC |

---

### 2.3 RTC 可访问状态枚举（匿名枚举）

| 枚举值 | 说明 |
|---|---|
| `RTC_UNACCESSIBLE` (0) | RTC 外设不可访问（未初始化或总线故障） |
| `RTC_ACCESSIBLE_TIME_UNRELIABLE` (1) | RTC 可读写，但时间不可信（如首次上电尚未同步） |
| `RTC_ACCESSIBLE_TIME_RELIABLE` (2) | RTC 可读写且时间可信（已通过网络或用户同步） |

---

### 2.4 `enum`（`rtc_app.h`）——唤醒源

| 枚举值 | 说明 |
|---|---|
| `R3_WKUP_SRC_ALM` (0) | 闹钟到点唤醒 |
| `R3_WKUP_SRC_256HZ` (4) | 256 Hz 周期唤醒（约每 3.9 ms 唤醒一次） |
| `R3_WKUP_SRC_64HZ` (5) | 64 Hz 周期唤醒（约每 15.6 ms） |
| `R3_WKUP_SRC_2HZ` (6) | 2 Hz 周期唤醒（约每 500 ms） |
| `R3_WKUP_SRC_1HZ` (7) | 1 Hz 周期唤醒（每 1 秒） |

---

## 三、核心接口（推荐直接使用）

### 3.1 `rtc_config_init`

```c
void rtc_config_init(const struct rtc_dev_platform_data *arg);
```

**功能：** 将平台配置数据绑定到 RTC 驱动函数表，完成驱动路由初始化。内部根据 `arg->rtc_sel` 决定后续调用硬件 RTC 还是虚拟 RTC 的底层实现。

**调用时机：** 由 SDK 设备管理框架在系统初始化时自动调用（`device_config.c` 注册设备表后），一般不需要手动调用。

---

### 3.2 `rtc_read_time`

```c
void rtc_read_time(struct sys_time *curr_time);
```

**功能：** 读取 RTC 当前时间，写入调用方提供的 `sys_time` 结构体。

**注意：**

- 硬件 RTC 直接读寄存器，结果为实时值。
- 虚拟 RTC 从 LRC 计数 + P11 RAM 快照推算当前时间。

**示例：**

```c
struct sys_time now = {0};
rtc_read_time(&now);
printf("%04d-%02d-%02d %02d:%02d:%02d\n",
       now.year, now.month, now.day, now.hour, now.min, now.sec);
```

---

### 3.3 `rtc_write_time`

```c
void rtc_write_time(const struct sys_time *curr_time);
```

**功能：** 将指定时间写入 RTC，立即生效，RTC 从此时刻开始计时。

**使用场景：**

- 网络对时后同步 RTC。
- 首次设置用户时间。
- 测试时写入已知时间点再读回验证。

**示例：**

```c
struct sys_time t = {
    .year = 2025, .month = 6, .day = 15,
    .hour = 10,   .min   = 30, .sec = 0,
};
rtc_write_time(&t);
```

---

### 3.4 `rtc_read_alarm`

```c
void rtc_read_alarm(struct sys_time *alarm_time);
```

**功能：** 读取当前设置的闹钟时刻，写入调用方提供的 `sys_time` 结构体。

---

### 3.5 `rtc_write_alarm`

```c
void rtc_write_alarm(const struct sys_time *alarm_time);
```

**功能：** 设置闹钟触发时刻。写入后还需调用 `rtc_alarm_en(1)` 才会使能闹钟。

**示例：**

```c
struct sys_time alarm = {
    .year = 2025, .month = 6, .day = 15,
    .hour = 10,   .min   = 30, .sec = 20,
};
rtc_write_alarm(&alarm);
rtc_alarm_en(1);
```

---

### 3.6 `rtc_alarm_en`

```c
void rtc_alarm_en(u8 set_alarm);
```

**功能：** 使能或禁用闹钟中断。

| 参数 | 说明 |
|---|---|
| `1` | 使能闹钟；当 RTC 时间到达 `rtc_write_alarm` 设置的时刻，触发 `cbfun` 回调 |
| `0` | 禁用闹钟；已触发的闹钟在回调中调用此函数可防止持续触发 |

---

### 3.7 `rtc_get_alarm_en`

```c
u32 rtc_get_alarm_en(void);
```

**功能：** 返回当前闹钟使能状态，非零表示闹钟已使能。

---

### 3.8 `rtc_debug_dump`

```c
void rtc_debug_dump(void);
```

**功能：** 将 RTC 相关的寄存器原始值和当前时间通过日志打印出来，供调试确认 RTC 外设是否正常初始化。

**使用场景：** 上电后调用，观察打印是否非全 0，确认 RTC 已正常初始化并持有默认时间。

---

### 3.9 `poweroff_save_rtc_time`

```c
void poweroff_save_rtc_time(void);
```

**功能：** 在软关机流程中调用，将当前 RTC 时间保存到 P11 掉电域（虚拟 RTC 需要，硬件 RTC 下为空操作或做快照备份）。

**调用位置：** `sys_enter_soft_poweroff()` 函数开头。

---

## 四、时间工具函数

### 4.1 `copy_time`

```c
void copy_time(struct sys_time *old_time, struct sys_time *new_time);
```

**功能：** 将 `old_time` 的全部字段浅拷贝到 `new_time`，等价于 `*new_time = *old_time`。

---

### 4.2 `time_diff_for_sec`

```c
u32 time_diff_for_sec(struct sys_time *old_time, struct sys_time *curr_time);
```

**功能：** 计算 `curr_time - old_time` 的秒数差（`curr_time` ≥ `old_time`），返回 `u32`。

**使用场景：** 验证 RTC 走时精度，如下例：

```c
u32 elapsed = time_diff_for_sec(&start_time, &now);
// elapsed == 60 表示过去了 60 秒
```

---

### 4.3 `time_add_sec`

```c
void time_add_sec(struct sys_time *time, s64 sec);
```

**功能：** 在 `time` 上加（或减，`sec` 可为负）指定秒数，结果写回 `time`，自动处理进位/借位（秒→分→时→日→月→年，跨月末、年末均正确处理，含闰年）。

**使用场景：** 计算闹钟时刻（当前时间 + N 秒），如：

```c
struct sys_time alarm;
copy_time(&now, &alarm);
time_add_sec(&alarm, 20);   // 20 秒后触发闹钟
rtc_write_alarm(&alarm);
rtc_alarm_en(1);
```

---

### 4.4 `datetime_to_sec`

```c
u64 datetime_to_sec(const struct sys_time *curr_time);
```

**功能：** 将 `sys_time` 结构体转换为从某固定起点（内部参考点，不是 Unix epoch）起的总秒数，返回 `u64`。

**使用场景：** 配合 `sec_to_datetime` 做往返验证，或用于时间区间运算。

---

### 4.5 `sec_to_datetime`

```c
void sec_to_datetime(u64 total_seconds, struct sys_time *curr_time);
```

**功能：** `datetime_to_sec` 的逆操作，将总秒数转回 `sys_time` 结构体。

---

### 4.6 `caculate_weekday_by_time`

```c
u8 caculate_weekday_by_time(struct sys_time *r_time);
```

**功能：** 根据 `r_time` 中的年月日计算星期几。

**返回值：** `0` = 周日，`1` = 周一，…，`6` = 周六。

**示例：**

```c
struct sys_time t = {.year = 2025, .month = 6, .day = 16};
u8 wd = caculate_weekday_by_time(&t); // wd == 1，周一
```

---

### 4.7 `month_to_day`

```c
u16 month_to_day(u16 year, u8 month);
```

**功能：** 计算指定年份中，某月最后一天是当年第几天（即 1 月 1 日到该月末的累计天数）。

**说明：** 内部会自动判断闰年，2 月在闰年返回 31+29=60，平年返回 31+28=59。

**示例：**

```c
u16 d = month_to_day(2024, 2); // d == 60，2024 年为闰年
```

---

### 4.8 `ymd_to_day`

```c
u16 ymd_to_day(const struct sys_time *time);
```

**功能：** 将 `sys_time` 中的年月日转换为该年内的第几天（1 月 1 日 = 第 1 天）。

---

### 4.9 `day_to_ymd`

```c
void day_to_ymd(u16 day, struct sys_time *sys_time);
```

**功能：** `ymd_to_day` 的辅助逆操作，将当年第 `day` 天转回 `sys_time` 的月日字段（年由调用方预先设置）。

---

## 五、底层驱动函数（一般不需要直接调用）

以下函数由 SDK 驱动框架内部使用，或在极少数需要直接操作寄存器时才会用到。

### 5.1 `rtc_init`

```c
int rtc_init(const struct rtc_dev_platform_data *arg);
```

**功能：** RTC 外设底层初始化，配置时钟源、写入默认时间和闹钟时刻、注册 cbfun。正常情况下由 `rtc_config_init` 间接调用，不需要手动调用。

---

### 5.2 `rtc_ioctl`

```c
int rtc_ioctl(u32 cmd, u32 arg);
```

**功能：** 统一的 RTC 控制接口，通过 `cmd` 指定操作类型（如读时间、写时间、设置闹钟等），供设备框架统一调度。

---

### 5.3 `rtc_wakup_source`

```c
void rtc_wakup_source(void);
```

**功能：** 查询并处理 RTC 唤醒源（对应 `rtc_app.h` 中的 `R3_WKUP_SRC_*` 枚举），在从深度睡眠或软关机唤醒后由系统调用，判断是闹钟唤醒还是周期定时唤醒。

---

### 5.4 `set_alarm_ctrl`

```c
void set_alarm_ctrl(u8 set_alarm);
```

**功能：** 直接操作 P33 域闹钟使能寄存器（`R3_ALM_CON` 的 `ALM_ALMEN` 位），是 `rtc_alarm_en` 的底层实现，推荐用 `rtc_alarm_en` 代替。

---

### 5.5 `write_sys_time` / `read_sys_time`

```c
void write_sys_time(struct sys_time *curr_time);
void read_sys_time(struct sys_time *curr_time);
```

**功能：** 直接读写 P33 域 RTC 时间寄存器（`R3_RTC_CON0` 控制读写使能后操作），是 `rtc_write_time` / `rtc_read_time` 的底层实现。

---

### 5.6 `write_alarm` / `read_alarm`

```c
void write_alarm(struct sys_time *alarm_time);
void read_alarm(struct sys_time *alarm_time);
```

**功能：** 直接读写 P33 域闹钟寄存器，是 `rtc_write_alarm` / `rtc_read_alarm` 的底层实现。

---

## 六、P11 域时间持久化接口

P11 掉电域在 VDD11 断电时仍能保持数据（通常由电池或超级电容维持），主要用于虚拟 RTC 场景下保存时间快照，在 `CLK_SEL_LRC` 模式下保证软关机后时间连续。

### 6.1 `p11_sys_time_init`

```c
void p11_sys_time_init(void);
```

**功能：** 初始化 P11 域时间存储，读取 P11 中已有的时间快照并恢复到当前 RTC，若 P11 数据无效则使用默认时间。由 `rtc_config_init` 内部调用。

---

### 6.2 `read_p11_sys_time`

```c
bool read_p11_sys_time(struct sys_time *t, struct _rtc_trim *lrc_trim);
```

**功能：** 从 P11 掉电域读取时间快照和 LRC 校准参数。

**返回值：** `true` 表示 P11 数据有效可用，`false` 表示数据无效（需使用默认时间）。

---

### 6.3 `write_p11_sys_time`

```c
int write_p11_sys_time(int param);
```

**功能：** 将当前时间写入 P11 掉电域，供下次上电恢复使用。由定时器周期调用（`write_p11_sys_time_by_timer`）或在软关机前调用。

---

### 6.4 `get_lrc_rtc_trim`

```c
void get_lrc_rtc_trim(struct _rtc_trim *lrc_trim);
```

**功能：** 获取当前 LRC 振荡器的校准参数，用于虚拟 RTC 的精度补偿计算。

---

## 七、PR 口（P33 域 GPIO）操作接口

P33 RTCVDD 域有一组专属的 PR 口（PR0–PRn），在系统主电源断开（软关机）后依然可以通过这些 GPIO 维持低功耗外设通信。

所有接口均返回 `int`（0 表示成功，负值表示错误），参数 `port` 为 PR 口编号。

| 函数 | 说明 |
|---|---|
| `rtc_port_pr_read(port)` | 读取 PR 口当前电平，返回 0 或 1 |
| `rtc_port_pr_out(port, value)` | 设置 PR 口输出电平（`value`: 0 = 低，1 = 高） |
| `rtc_port_pr_dir(port, dir)` | 设置 PR 口方向（`dir`: 0 = 输出，1 = 输入） |
| `rtc_port_pr_die(port, die)` | 设置 PR 口数字输入使能（`die`: 1 = 使能数字输入） |
| `rtc_port_pr_pu(port, value)` | 设置 PR 口上拉（`value`: 1 = 使能上拉） |
| `rtc_port_pr_pu1(port, value)` | 设置 PR 口强上拉（`value`: 1 = 使能） |
| `rtc_port_pr_pd(port, value)` | 设置 PR 口下拉（`value`: 1 = 使能下拉） |
| `rtc_port_pr_pd1(port, value)` | 设置 PR 口强下拉（`value`: 1 = 使能） |
| `rtc_port_pr_hd0(port, value)` | 设置 PR 口驱动能力 0 档（`value`: 1 = 使能） |
| `rtc_port_pr_hd1(port, value)` | 设置 PR 口驱动能力 1 档（`value`: 1 = 使能） |

---

## 八、寄存器操作宏（`rtc_app.h`，底层使用）

这些宏直接操作 P33 域寄存器，封装了 `P33_CON_SET`，通常只在驱动底层使用，应用层不需要直接调用。

### 8.1 R3_ALM_CON——闹钟控制寄存器

| 宏 | 位域 | 说明 |
|---|---|---|
| `ALM_ALMOUT(a)` | bit7, 1 bit | 闹钟输出引脚控制，`1` = 闹钟触发时拉高输出引脚 |
| `ALM_CLK_SEL(a)` | bit[4:2], 3 bit | 闹钟时钟源选择，与 `enum RTC_CLK` 对应 |
| `ALM_ALMEN(a)` | bit0, 1 bit | 闹钟中断使能，`1` = 使能，`0` = 禁用 |

### 8.2 R3_RTC_CON0——RTC 读写控制寄存器

| 宏 | 位域 | 说明 |
|---|---|---|
| `RTC_ALM_RDEN(a)` | bit5 | 闹钟寄存器读使能，读操作前置 1，完成后清 0 |
| `RTC_RTC_RDEN(a)` | bit4 | RTC 时间寄存器读使能 |
| `RTC_ALM_WREN(a)` | bit1 | 闹钟寄存器写使能，写操作前置 1 |
| `RTC_RTC_WREN(a)` | bit0 | RTC 时间寄存器写使能 |

### 8.3 R3_OSL_CON——32K 晶振控制寄存器

| 宏 | 位域 | 说明 |
|---|---|---|
| `OSL_X32XS(a)` | bit[5:4], 2 bit | 32K 晶振内部负载电容选择（0–3 档，对应 `rtc_dev_platform_data.x32xs`） |
| `OSL_X32TS(a)` | bit2 | 32K 晶振测试模式，通常为 0 |
| `OSL_X32OS(a)` | bit1 | 32K 晶振输出使能 |
| `OSL_X32ES(a)` | bit0 | 32K 晶振使能，使用外部晶振时须为 1 |

### 8.4 R3_TIME_CPND——周期定时中断标志清除

写 1 清除对应中断挂起标志，在中断处理函数中使用。

| 宏 | 说明 |
|---|---|
| `TIME_256HZ_CPND(a)` | 清除 256 Hz 定时中断标志 |
| `TIME_64HZ_CPND(a)` | 清除 64 Hz 定时中断标志 |
| `TIME_2HZ_CPND(a)` | 清除 2 Hz 定时中断标志 |
| `TIME_1HZ_CPND(a)` | 清除 1 Hz 定时中断标志 |

---

## 九、应用封装层：`rdx_rtc`（`rdx_rtc.h`）

`rdx_rtc` 是在 `rtc_dev.h` 之上的应用级封装，提供以 Unix 时间戳（`time_t`）为中心的接口，并同时支持硬件 RTC 和软件模拟 RTC（通过宏 `RDX_RTC_USE_HW` 切换）。

**切换宏：**

```c
// rdx_rtc.h
#define RDX_RTC_USE_HW  1   // 1 = 硬件 RTC；0 = 软件模拟（jiffies + Flash）
```

### 9.1 初始化与开关机

| 函数 | 说明 |
|---|---|
| `rdx_rtc_init(void)` | 初始化 RTC：硬件模式下检查年份有效性，年份 < 2025 则重置为默认时间；软件模式下从 Flash 恢复基准值并启动定时持久化 |
| `rdx_rtc_print_boot(void)` | 开机时打印 RTC 当前时间和模式标签（`[HW]`/`[SW]`），与 `rdx_rtc_print_poweroff` 配对用于验证软关机后时间是否连续 |
| `rdx_rtc_print_poweroff(void)` | 软关机前调用，确保时间持久化后打印关机时刻 |

### 9.2 时间读写

| 函数 | 说明 |
|---|---|
| `rdx_rtc_get(void) → time_t` | 获取当前 Unix 时间戳：硬件模式从 `rtc_read_time` 读取并转换；软件模式从 Flash + jiffies 偏移推算 |
| `rdx_rtc_set_timestamp(time_t) → int` | 设置 RTC：硬件模式将时间戳转为 `sys_time` 后调用 `rtc_write_time`；软件模式写入 Flash 并更新 jiffies 基准 |
| `rdx_rtc_store_timestamp(void)` | 将当前时间持久化：软件模式写 Flash，硬件模式为空操作（P33 RTCVDD 域掉电不丢时间） |
| `rdx_rtc_restore_timer_start(void)` | 启动 Flash 持久化定时器（每 3 分钟保存一次，软件模式专用，硬件模式为空操作） |
| `rdx_rtc_restore_timer_stop(void)` | 停止 Flash 持久化定时器（软件模式专用） |

### 9.3 字符串与格式化

| 函数 | 返回值 | 说明 |
|---|---|---|
| `rdx_rtc_get_string(void)` | `char *`（静态缓冲区） | 返回当前时间字符串，格式 `"YYYY-MM-DD HH:MM:SS"` |
| `rdx_rtc_get_string_date(void)` | `char *`（静态缓冲区） | 返回当前日期字符串，格式 `"YYYY-MM-DD"` |
| `rdx_rtc_get_string_time(void)` | `char *`（静态缓冲区） | 返回当前时间字符串，格式 `"HH:MM:SS"` |
| `rdx_rtc_timestamp_to_utc_string(time_t, char*)` | void | 将时间戳转为 UTC 字符串写入 `buffer` |
| `rdx_rtc_timestamp_to_timezone_string(time_t, int, char*)` | void | 将 UTC 时间戳加上时区偏移（秒，如 `+28800` 表示 UTC+8）后写入 `buffer` |

> **注意：** `rdx_rtc_get_string*` 返回的是函数内静态缓冲区，非线程安全，调用后需立即使用或拷贝。

### 9.4 时间戳与 DateTime 互转

```c
typedef struct {
    int year, month, day, hour, minute, second;
    int weekday;           // 0=Sunday … 6=Saturday
    const char *weekday_name;
} DateTime;
```

| 函数 | 说明 |
|---|---|
| `rdx_rtc_timestamp_to_datetime(time_t) → DateTime` | Unix 时间戳 → `DateTime` 结构体，含星期字段和星期名称字符串 |
| `rdx_rtc_datetime_to_timestamp(DateTime) → time_t` | `DateTime` 结构体 → Unix 时间戳 |
| `rdx_rtc_utc_string_to_timestamp(const char*) → time_t` | 格式为 `"YYYY-MM-DD HH:MM:SS"` 的字符串 → Unix 时间戳 |
| `rdx_rtc_parse_utc_string(const char*, int*, …)` | 解析 `"YYYY-MM-DD HH:MM:SS"` 字符串为各个整数字段（内部辅助函数） |

### 9.5 日历辅助

| 函数 | 说明 |
|---|---|
| `rdx_rtc_is_leap_year(int year) → int` | 判断是否为闰年，是返回 1，否返回 0（规则：能被 4 整除且不被 100 整除，或能被 400 整除） |
| `rdx_rtc_days_in_month(int year, int month) → int` | 返回指定年月的天数，闰年 2 月返回 29 |
| `rdx_rtc_calculate_weekday(int year, int month, int day) → int` | 基于 Zeller 公式计算星期几，`0`=周六，`1`=周日，…，`6`=周五（Zeller 原始输出，与 `caculate_weekday_by_time` 定义不同，注意区分） |

### 9.6 测试

| 函数 | 说明 |
|---|---|
| `rdx_rtc_test(void) → int` | 打印当前 RTC 时间、时间戳及 `DateTime` 各字段，供调试用 |

## 十一、快速使用示例

### 读取当前时间

```c
#include "rtc/rtc_dev.h"

struct sys_time now = {0};
rtc_read_time(&now);
printf("%04d-%02d-%02d %02d:%02d:%02d\n",
       now.year, now.month, now.day, now.hour, now.min, now.sec);
```

### 设置时间

```c
struct sys_time t = {.year=2025,.month=6,.day=15,.hour=10,.min=0,.sec=0};
rtc_write_time(&t);
```

### 20 秒后触发闹钟

```c
struct sys_time now, alarm;
rtc_read_time(&now);
copy_time(&now, &alarm);
time_add_sec(&alarm, 20);
rtc_write_alarm(&alarm);
rtc_alarm_en(1);
// 闹钟到点后 device_config.c 中注册的 cbfun 会被调用
// 在回调中记得 rtc_alarm_en(0) 关闭闹钟
```

### 计算两个时刻的间隔

```c
struct sys_time t_start = {.year=2025,.month=6,.day=15,.hour=10,.min=0,.sec=0};
struct sys_time t_end   = {.year=2025,.month=6,.day=15,.hour=10,.min=1,.sec=30};
u32 diff = time_diff_for_sec(&t_start, &t_end); // diff == 90
```

### 获取星期几

```c
struct sys_time t = {.year=2025,.month=6,.day=16}; // 周一
u8 wd = caculate_weekday_by_time(&t); // wd == 1
```

### 通过时间戳操作 RTC（rdx_rtc 封装）

```c
#include "rdx_rtc.h"

// 初始化（在 app 初始化流程中调用一次）
rdx_rtc_init();

// 读取当前时间戳
time_t ts = rdx_rtc_get();

// 获取格式化字符串
printf("%s\n", rdx_rtc_get_string()); // "2025-06-15 10:30:00"

// 通过 UTC 字符串设置时间
rdx_rtc_set_timestamp(rdx_rtc_utc_string_to_timestamp("2025-06-15 10:30:00"));

// 软关机前
rdx_rtc_print_poweroff();
```
