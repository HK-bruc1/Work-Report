## 3. 框图

MK SD NAND 是一种以 LGA 封装形式设计的嵌入式存储解决方案，其工作方式与行业标准 SD 卡类似。SD NAND 由 NAND 闪存和一颗高性能控制器组成，支持超高速模式（UHS-I），完全符合 SD 2.0 接口标准。凭借卓越的读写性能、可靠性和主控兼容性，MK SD NAND 旨在为面向 SD 的应用提供高性能、低功耗、高性价比且增值的存储解决方案。

## 5. 引脚分配（SD模式与SPI模式）

| 引脚 | 名称 | 类型 | SD 模式 | SPI 模式 |
|------|------|------|---------|----------|
| 1 | DAT2 | I/O/PP | 数据线 2 | 保留 (RSV) |
| 2 | CD/DAT3 | I/O/PP | 卡检测 / 数据线 3 | 片选信号 (CS) |
| 3 | CMD | PP | 命令/响应线 | 数据输入 (DI) |
| 4 | VDD | S | 电源 | 电源 |
| 5 | CLK | I | 时钟 | 时钟 (SCLK) |
| 6 | VSS | S | 地 | 地 |
| 7 | DAT0 | I/O/PP | 数据线 0 | 数据输出 (DO) |
| 8 | DAT1 | I/O/PP | 数据线 1 | 保留 (RSV) |

> **引脚类型说明**：
> - S = 电源
> - I = 输入
> - O = 推挽驱动器输出
> - PP = 推挽驱动器 I/O

**顶部视图**：
```
┌──────────────────────────┐
│  DAT2  CD/DAT3  CMD  VDD │
│   (1)    (2)    (3)  (4) │
│                          │
│   CLK    VSS   DAT0  DAT1│
│   (5)    (6)   (7)   (8) │
└──────────────────────────┘
```

**注意事项**：
- 扩展数据引脚（DAT1-DAT3）在上电时为输入。在 SET_BUS_WIDTH 命令后，它们开始作为数据引脚工作。
- 上电时，此引脚在卡内启用了50kΩ的上拉电阻。该电阻有两个功能：卡检测和模式选择。
  - **模式选择**：主机可以将该引脚驱动为高电平或让其被上拉高电平以选择 SD 模式。如果主机想选择 SPI 模式，应将引脚驱动为低电平。
  - **卡检测**：主机检测到该引脚被上拉高电平。在常规数据传输期间，应通过 SET_CLR_CARD_DETECT (ACMD42) 命令断开此上拉电阻。

## 6. 使用方法

### 6.1 SD 总线模式协议

SD 总线允许将双向数据信号线的数量从1条动态配置到4条。上电后默认情况下，SD卡仅使用 DAT0。初始化后，主机可以更改总线宽度。

支持多片 SD 卡连接到主机。在多卡连接中，VDD、VSS 和 CLK 信号可共用。但命令（Command）、响应（Response）和数据线（DAT0-DAT3）需要主机为每个设备单独分配。

该特性便于在硬件成本和系统性能之间进行权衡。SD 总线上的通信基于由起始位发起、由停止位终止的命令和数据位流。

**命令 (Command)**

命令在 CMD 线上串行传输。命令是从主机到设备的操作令牌。命令可以发送给单个寻址的卡（寻址命令）或所有已连接的卡（广播命令）。

**响应 (Response)**

响应在 CMD 线上串行传输。响应是对之前接收到的命令的回答。响应从单个寻址的卡或所有已连接的卡发出。

**数据 (Data)**

数据可以从卡传输到主机，也可以从主机传输到卡。数据通过数据线传输。

**SD NAND 信号说明**：

| 信号 | 说明 |
|------|------|
| CLK | 主机卡时钟信号 |
| CMD | 双向命令/响应信号 |
| DAT0-DAT3 | 4位双向数据信号 |
| VDD | 电源 |
| VSS | 地 |

### 6.2 卡初始化

要初始化 SD NAND，建议按照以下步骤进行：

1. **提供初始化电压**
   
   主机系统可以从初始化时开始向卡施加工作电压。向 SD 卡发送至少74个周期的虚拟时钟。

2. **选择操作模式（SD 模式或 SPI 模式）**
   
   - **SPI 模式操作**：主机应将 SD 卡接口的1引脚（CD/DAT3）驱动为"低"电平，然后发送 CMD0。
   - **SD 模式操作**：主机应驱动或检测 SD 卡接口的1引脚（正常上拉为高电平）。
   
   卡保持选定的操作模式，除非重新发送 CMD0 或重新上电。

**以下为 SD 模式初始化流程**：

3. 发送 ACMD41（参数 Arg=0），识别卡的工作电压范围。

4. 向卡施加指示的工作电压。
   
   重新发送 ACMD41 并应用已存储的电压，重复 ACMD41 直到忙位被清除（位31 忙=1）。如果响应超时，主机可判断不是 SD 卡。

5. 发送 CMD2，获取卡 ID（CID）。

6. 发送 CMD3，获取 RCA。（RCA 值每次访问随机变化，不等于零）

7. 发送 CMD7，进入传输状态。
   
   如有必要，主机可以发送 ACMD42 并禁用用于卡检测的上拉电阻。

8. 发送 ACMD13，轮询卡状态作为 SD 存储卡。检查 SD_CARD_TYPE 值。如果有效8位全为"零"，则表示 SD 卡。如果不是，则停止初始化。

9. 发送 CMD7 进入待机状态。发送 CMD9 获取 CSD。发送 CMD10 获取 CID。

10. 用 CMD7 返回传输状态。

11. 发送 ACMD6，选择合适的总线宽度。

然后主机可以将 SD 卡作为存储设备访问数据。

## 7. 内部信息

### 7.1 寄存器

SD NAND 拥有六个寄存器和 SD 状态信息：OCR、CID、CSD、RCA、DSR、SCR 和 SD 状态。

本卡**不支持 DSR**。

寄存器分为两类：

- **MMC 兼容寄存器**：OCR、CID、CSD、RCA、DSR、SCR
- **SD 卡专用寄存器**：SD Status

**SD 卡寄存器汇总**：

| 寄存器名称 | 位宽 | 说明 |
|-----------|------|------|
| OCR | 32 | 操作条件（VDD 电压配置和忙状态） |
| CID | 128 | 卡识别信息 |
| CSD | 128 | 卡特定信息 |
| RCA | 16 | 相对卡地址 |
| DSR | 16 | 未实现（可编程卡驱动器）：驱动级寄存器 |
| SCR | 64 | SD 存储卡特殊功能 |
| SD Status | 512 | 状态位和卡功能 |

#### 7.1.1 OCR 寄存器

这个32位寄存器描述了工作电压范围和电源供电状态位。

**OCR 寄存器定义**：

| OCR 位 | VDD 电压窗口 | 初始值 |
|--------|-------------|--------|
| 31 | 卡上电状态位（忙） - "0" = 忙 | — |
| 30 | 卡容量状态 - "0" = SD 存储卡 | — |
| 29-25 | 保留 | 全0 |
| 24 | 切换至1.8V 接受 (S18A) | 0 |
| 23 | 3.6-3.5V | 1 |
| 22 | 3.5-3.4V | 1 |
| 21 | 3.4-3.3V | 1 |
| 20 | 3.3-3.2V | 1 |
| 19 | 3.2-3.1V | 1 |
| 18 | 3.1-3.0V | 1 |
| 17 | 3.0-2.9V | 1 |
| 16 | 2.9-2.8V | 1 |
| 15 | 2.8-2.7V | 1 |
| 14-8 | 保留 | 0 |
| 7-4 | 保留（用于低电压范围） | 0 |
| 3-0 | 保留 | 全0 |

## 8. 供电方案

### 8.1 上电

"上电时间"定义为从0伏电压上升到VDD最小值的电压上升时间。

"电源爬升时间"提供了电源建立到工作电平（主机供电电压）的时间，以及等待 SD NAND 可以接受第一条命令的时间。

主机应向卡供电，使电压在250ms内达到Vdd_min，并在保持CMD线为高的同时，向SD NAND提供至少74个SD时钟。

### 8.2 上电时间

主机需要在上电爬升前将电源线电平保持在0.5V以下至少1ms。

#### 8.2.1 上电或电源循环

以下是上电和电源循环的要求，以确保可靠的 SD NAND 硬件复位：

1. 电压电平应低于0.5V
2. 持续时间应至少为1ms

#### 8.2.2 电源爬升

电源爬升时间定义为从0.5V阈值电平上升到工作供电电压（在VDD(最小值)和VDD(最大值)之间稳定），此时主机可以供应SD时钟。

**电源爬升的建议**：

1. 电源爬升电压应尽可能单调
2. 最小爬升时间应为0.1ms
3. 对于2.7-3.6V电源，最大爬升时间应为35ms

#### 8.2.3 电源下降

当主机关闭电源时，VDD 应在至少1ms的时间内降低到0.5V以下。在断电期间，DAT、CMD 和 CLK 应由主机断开或驱动到逻辑0，以避免通过信号线吸取工作电流的情况。

如果主机需要更改工作电压，则需要电源循环。电源循环意味着电源被关闭并重新供电。对于访问已处于非活动状态的卡也需要电源循环。要进行电源循环，主机应在重新给卡上电之前遵循断电说明（即VDD应至少降低到0.5V以下至少1ms）。

## 11. 参考设计

### 推荐电路连接

```
        VDD (3.3V)
          │
          ├──[2.2μF]──┬────────────── VDD
          │            │
        VSS ──────────┴────────────── VSS
          │
        CLK ──[0~120Ω, 推荐22Ω]── CLK
          │
        CMD ──[10K~100KΩ]上拉── CMD
          │
       DAT0 ──[10K~100KΩ]上拉── DAT0
       DAT1 ──[10K~100KΩ]上拉── DAT1
       DAT2 ──[10K~100KΩ]上拉── DAT2
       DAT3 ──[10K~100KΩ]上拉── DAT3
```

> **注意事项**：
> 1. **RDAT 和 RCMD（10K~100KΩ）** 是上拉电阻，用于在SD NAND处于高阻态时保护CMD和数据线免受总线浮空的影响。
> 2. 即使主机在SD模式下仅以1位模式使用SD NAND，主机也应通过RDAT将所有DAT0-3引脚上拉。
> 3. 建议在VDD上配置2.2μF的电容。
> 4. **RCLK 参考值 0~120Ω，推荐 22Ω**。
