# 存储类芯片标准 SPI 的硬件设计

## —— 单线复用半双工模式的适用场景与芯片类型论证

---

## 1. 设计背景与动机

### 1.1 引脚资源困境

在嵌入式系统设计中，MCU 的 GPIO 引脚往往是最稀缺的资源。以 TWS 耳机充电仓、智能手环、微型传感器等场景为例：

- **BR28 平台**：仅 20+ 个可用 GPIO，需同时承载充电管理、LED 指示、按键检测、蓝牙通信等功能
- **每省 1 个 IO 可为其他功能留出设计余量**：单线方案将 SPI 从 4 线（CS/CLK/SI/SO）压缩到 3 线（CS/CLK/DIO），节省的 1 个 GPIO 可用于电池电量检测或扩展传感器

### 1.2 通信本质分析

存储类芯片的 SPI 通信具有显著的**主从单向交替特征**：

| 操作阶段 | 主机行为 | 从机行为 | 数据流向 |
|---------|---------|---------|---------|
| 发命令 | 驱动电平 | 监听 | MCU → Flash |
| 发地址 | 驱动电平 | 监听 | MCU → Flash |
| 写数据 | 驱动电平 | 接收写入 | MCU → Flash |
| 读数据 | 释放总线（输入） | 驱动电平 | Flash → MCU |

**关键洞察**：虽然标准 SPI 有独立的 SI/SO 线，但在标准单线 SPI 命令/地址/数据阶段，存储芯片**不会同时发送和接收**。从机（Flash）仅在主机请求读数据时才驱动 SO 线，其余时间 SO 处于高阻态（Hi-Z）。这一特性为单线复用提供了物理基础。

## 2. 核心硬件方案：单线 Composite 双向数据线

### 2.1 电路拓扑

```
MCU (SPI_HOST)                       存储芯片 (SPI_SLAVE)
┌──────────────┐                     ┌──────────────┐
│              │                     │              │
│     DIO ─────┼──────[330Ω]────┬────┼── SI (输入)   │
│              │                │    │              │
│              │                │    │              │
│              │                │    │              │
│              │                │    │              │
│              │                └────┼── SO (输出)   │
│              │                     │              │
│    SCLK ─────┼────────────────────┼── SCLK        │
│     CS# ─────┼────────────────────┼── CS#         │
└──────────────┘                     └──────────────┘
```

**元件清单**：
- 1 颗 330Ω 贴片电阻（0603/0402，成本 < ¥0.001）
- 无需额外模拟开关或方向控制 GPIO

### 2.2 工作原理：分时半双工

| 阶段 | MCU GPIO 方向 | Flash SI 状态 | Flash SO 状态 | 总线状态 | 数据流向 |
|------|--------------|---------------|---------------|---------|---------|
| 发命令/地址 | 输出（推挽驱动） | 监听输入 | **高阻态 (Hi-Z)** | MCU 独占 | MCU → Flash |
| 写数据 | 输出（推挽驱动） | 接收数据 | **高阻态 (Hi-Z)** | MCU 独占 | MCU → Flash |
| 读数据 | 输入（高阻态） | 忽略 | **推挽驱动** | Flash 独占 | Flash → MCU |

**330Ω 电阻的作用**：
- 在方向切换的**极短瞬间**（ns 级），若 MCU 和 Flash 同时驱动不同电平，电阻限制短路电流至 < 10mA（3.3V / 330Ω），保护引脚
- 正常工作时，330Ω 串联电阻与负载输入电容形成 RC 低通滤波，边沿退化量约为：$t_{rise\_degradation} \approx 2.2 \times 330\Omega \times C_{load}$。以典型 C_load = 10pF 计，$t_r \approx 7.3\text{ns}$，在 50MHz（周期 20ns）下仍在多数芯片的时序容限内。但需注意 330Ω 约为典型传输线特征阻抗（50Ω）的 6.6 倍，在方向切换时会引入阻抗不连续点，存在信号反射风险。建议 >25MHz 应用降至 220Ω，并尽量缩短走线

## 3. 双芯片实例论证

### 3.1 实例一：GD5F4GM7（SPI NAND Flash）—— 已验证通过

**芯片参数**：
- 厂商：GigaDevice（兆易创新）
- 容量：4Gb（512MB）
- 接口：标准 SPI + Dual SPI + Quad SPI
- 封装：WSON-8

**硬件连接**：

```
BR28 MCU                           GD5F4GM7
┌──────────┐                     ┌──────────┐
│          │                     │          │
│ DIO(PA1)─┼──────[330Ω]────┬────┼─ SI(引脚5)│
│          │                │    │          │
│          │                │    │          │
│          │                │    │          │
│          │                │    │          │
│          │                └────┼─ SO(引脚2)│
│          │                     │          │
│ SCLK(PA3)┼─────────────────────┼─ SCLK    │
│ CS#(PA0)─┼─────────────────────┼─ CS#(引脚1)│
└──────────┘                     └──────────┘
```

**验证结果**：
- 平台：BR28（蓝牙音频 SoC）
- 模式：`SPI_MODE_UNIDIR_1BIT`（半双工）
- 操作：成功初始化、读取 Flash ID、正常读写数据
- 稳定性：长时间运行无异常

**牺牲的能力**：
- Dual SPI (x2) 和 Quad SPI (x4) 无法使用（SI/SO 共享同一数据线，无法独立驱动）
- `spi_read_width` 必须固定为 `1`

### 3.2 实例二：XCSD02G-MJ（SD NAND Flash）—— 理论可行，待验证

**芯片参数**（来源：芯存科技规格书）：
- 厂商：深圳市芯存科技有限公司
- 容量：2Gbit（256MB）
- 接口：SD 4-bit 模式 / SPI 模式
- 封装：LGA-8（8×6mm）
- 工作电压：2.7V ~ 3.6V
- 最高频率：50MHz

**SPI 模式引脚映射**：

| LGA-8 引脚 | SD 模式 | SPI 模式 | 类型 | 单线方案连接 |
|-----------|--------|---------|------|------------|
| 1 | DAT2 | RSV（保留） | - | 悬空 |
| 2 | CD/DAT3 | **CS#** | 输入 | MCU CS# |
| 3 | SCLK | **SCLK** | 输入 | MCU SCLK |
| 4 | Vss | Vss | 电源 | GND |
| 5 | CMD | **DI（SI）** | 输入 | 复用 Data 线 |
| 6 | DAT0 | **SO（DO）** | O/PP | 复用 Data 线 |
| 7 | DAT1 | RSV（保留） | - | 悬空 |
| 8 | Vcc | Vcc | 电源 | 3.3V + 2.2uF |

**关键差异与适配分析**：

| 对比项 | GD5F4GM7 | XCSD02G-MJ | 影响评估 |
|--------|----------|------------|---------|
| SO 三态特性 | 明确 Hi-Z | 标注 O/PP（推挽输出） | ⚠️ **需确认** DAT0 非读阶段是否释放总线 |
| 上拉电阻要求 | 无特殊要求 | **强制 10KΩ 上拉 DAT0** | ⚠️ 可能与 MCU 低电平驱动冲突 |
| 初始化流程 | 标准 SPI 命令 | **需先 74 时钟 + CMD0 进入 SPI 模式** | ⚠️ 时序更复杂 |
| 协议层 | 纯 SPI 命令集 | SD 协议子集（SPI 模式） | 驱动代码需适配 |

**硬件连接方案**：

```
MCU (SPI_HOST)                       XCSD02G-MJ (SD NAND)
┌──────────────┐                     ┌──────────────┐
│              │                     │              │
│     DIO ─────┼──────[330Ω]────┬────┼── CMD (引脚5) │  ← DI/SI
│              │                │    │              │
│              │                │    │              │
│              │                │    │              │
│              │                │    │              │
│              │                └────┼── DAT0(引脚6) │  ← SO（从机输出）
│              │                     │     ↑        │
│              │                     │   10KΩ 上拉   │  ← 规格书要求
│              │                     │     ↑        │
│              │                     │   3.3V       │
│              │                     │              │
│    SCLK ─────┼────────────────────┼── CLK (引脚3) │
│     CS ──────┼────────────────────┼── DAT3(引脚2) │  ← SPI 模式选择
└──────────────┘                     └──────────────┘
```

**上拉电阻冲突解决方案**：

方案 A（推荐）：**降低上拉电阻阻值**
- 将 10KΩ 改为 **4.7KΩ**，与 MCU 推挽输出形成分压
- MCU 低电平驱动能力通常 > 8mA，可可靠拉低 4.7KΩ 上拉
- 验证：V_low = 3.3V × R_MCU / (R_MCU + 4.7K)，典型值 < 0.3V

方案 B：**开漏输出 + 外部上拉**
- MCU 配置 GPIO 为开漏模式
- 复用 10KΩ 上拉作为总线上拉
- 牺牲部分驱动速度，但节省元件

**待验证项**：
1. 上电初始化阶段，DAT0 是否在 CMD 发送时保持高阻态
2. 10KΩ 上拉与 MCU 低电平驱动的兼容性（示波器实测）
3. SD NAND SPI 模式下的命令集兼容性（CMD0/ACMD41 等）

> **⚠️ 适用性判定**：XCSD02G-MJ 的 DAT0 引脚标注为 O/PP（推挽输出），与单线复用方案的核心前提——**SO 引脚必须在非读阶段处于高阻态（Hi-Z）**——存在潜在冲突。若实测确认 DAT0 在全阶段均为推挽驱动（不释放总线），则**不可使用单线复用方案**，必须采用标准双线 SPI。建议在采用单线方案前，完成上述 3 项待验证测试。

## 4. 适用场景论证

### 4.1 强烈推荐的场景（收益 > 风险）

| 场景 | 典型产品 | 引脚压力 | 速度需求 | 推荐度 |
|------|---------|---------|---------|--------|
| **TWS 耳机充电仓** | 苹果 AirPods 类 | 极高（20 引脚 MCU） | 低（配置数据） | ⭐⭐⭐⭐⭐ |
| **智能手环/手表** | 小米手环类 | 极高（QFN-32 封装） | 低（固件 + 日志） | ⭐⭐⭐⭐⭐ |
| **微型传感器节点** | 温湿度/气压传感器 | 高（需预留 ADC/GPIO） | 极低（周期性上报） | ⭐⭐⭐⭐⭐ |
| **电子烟控制板** | 调压/温控方案 | 高（需驱动 OLED/LED） | 低（用户配置） | ⭐⭐⭐⭐ |
| **玩具/教育硬件** | 儿童智能玩具 | 中（成本敏感） | 低（音频/图片） | ⭐⭐⭐⭐ |

### 4.2 可以使用的场景（需权衡）

| 场景 | 考量因素 | 建议 |
|------|---------|------|
| **工业控制模块** | 可靠性优先，引脚相对充裕 | 可用，但建议保留标准双线作为备选 |
| **智能家居控制器** | 需兼顾 OTA 升级速度 | 可用，OTA 时速度降低 50% 可接受 |
| **车载电子辅助模块** | 温度范围宽，信号完整性要求高 | 谨慎使用，需加强 EMC 设计 |

### 4.3 不建议使用的场景（风险 > 收益）

| 场景 | 原因 | 替代方案 |
|------|------|---------|
| **需要 Quad SPI 高速读取** | 单线方案永久禁用 x2/x4 模式 | 标准双线或四线 SPI |
| **XIP（就地执行）代码存储** | 读取延迟增加，影响 CPU 取指效率 | 标准双线 SPI 或并行 NOR |
| **大数据量连续传输**（> 10MB/s） | 单线带宽瓶颈（理论最大 6.25MB/s@50MHz） | SD 4-bit 模式或 eMMC |
| **多从机共享总线** | 方向切换时序复杂，易冲突 | 标准双线 + 独立 CS |
| **对信号完整性要求极高的高速设计** | 330Ω 电阻引入阻抗不连续 | 标准双线或差分信号 |

## 5. 适用芯片类型论证

### 5.1 芯片类型适用矩阵

| 芯片类型 | 代表型号 | SI/SO 独立性 | SO 三态支持 | 上拉要求 | 单线适用性 | 备注 |
|---------|---------|-------------|------------|---------|----------|------|
| **SPI NAND Flash** | GD5F4GM7, W25N01GV | ✅ 独立 | ✅ 明确 Hi-Z | 无特殊 | ⭐⭐⭐⭐⭐ | 最典型场景 |
| **SPI NOR Flash** | W25Q128, GD25Q64 | ✅ 独立 | ✅ 明确 Hi-Z | 无特殊 | ⭐⭐⭐⭐⭐ | 广泛适用 |
| **SD NAND (SPI 模式)** | XCSD02G-MJ, MKDV1GCL | ⚠️ 功能复用 | ⚠️ 需确认 | ⚠️ 强制上拉 | ⭐⭐⭐⭐ | 需验证 DAT0 三态 |
| **eMMC (SPI 模式)** | 部分兼容型号 | ❌ 不支持 | - | - | ❌ | eMMC 无 SPI 模式 |
| **SPI EEPROM** | AT25M01, BR25G128 | ✅ 独立 | ✅ 明确 Hi-Z | 无特殊 | ⭐⭐⭐⭐⭐ | 低速场景完美适配 |
| **SPI FRAM** | FM25W256 | ✅ 独立 | ✅ 明确 Hi-Z | 无特殊 | ⭐⭐⭐⭐⭐ | 耐久性高，适合单线 |
| **SPI SRAM** | 23LC1024 | ✅ 独立 | ✅ 明确 Hi-Z | 无特殊 | ⭐⭐⭐⭐ | 注意读写时序 |

### 5.2 关键判断标准

**可以使用单线复用的必要条件**：

1. **SO 引脚必须支持高阻态（Hi-Z）**
   - 数据手册中 SO 类型标注为 "O" 或 "O/PP" 且明确说明 "High-Z when not selected"
   - 反例：某些老型号 SPI 器件 SO 为推挽输出且持续驱动，**不可使用**

2. **SI 和 SO 在电气上独立**
   - 不能是内部短接的引脚（如某些简化封装）
   - 可以通过万用表测量 SI/SO 间阻值确认（应 > 1MΩ）

3. **通信协议为半双工交替**
   - 存储类芯片天然满足（命令 → 地址 → 数据，单向流动）
   - 反例：需要同时双向传输的通信芯片（如 SPI 以太网控制器）**不可使用**

4. **速度需求在单线带宽范围内**
   - 理论物理层速率：`Bandwidth_raw = Clock_Freq × 1bit / 8`（Byte/s @ 原始比特率）
   - @50MHz：物理层最大 6.25 MB/s，扣除命令/地址/等待周期开销后，有效带宽约 4~5 MB/s（利用率约 60%~80%）
   - 若应用需求 < 2 MB/s，单线方案完全胜任

## 6. 硬件设计规范

### 6.1 电阻选型指南

| 参数 | 推荐值 | 范围 | 说明 |
|------|--------|------|------|
| 串联电阻 R | **330Ω** | 220Ω ~ 1KΩ | 平衡保护能力与信号完整性 |
| 上拉电阻 R_pull | 4.7KΩ ~ 10KΩ | 1KΩ ~ 100KΩ | 需与 MCU 驱动能力匹配 |
| 电阻功率 | 1/16W (0402) | ≥ 1/20W | 短路瞬间功率 < 0.05W |
| 电阻精度 | ±5% | ±5% ~ ±10% | 无需高精度 |

**电阻位置**：
- 330Ω 串联在 DIO 走线上即可，限流保护功能与位置无关
- **优先靠近从机侧（Flash）放置**：SI/SO 的物理交汇点在 Flash 引脚附近，电阻靠近 Flash 可使 junction 到 SI/SO 焊盘的 stub 长度最短，同时避免 MCU 高阻抗源驱动长走线导致的边沿退化
- 注意：330Ω 的功能是**限流保护**，并非源端阻抗匹配（真正的源端匹配电阻为 22~33Ω，需靠近驱动端）。因此经典高速设计中"串联电阻靠源端"的准则在此不适用
- 若 PCB 布线受限，放在走线中间亦可接受（<50MHz 且走线 <5cm 时差异甚微）
- 若从机有强制上拉（如 XCSD02G-MJ），电阻位置不变

### 6.2 PCB 布局要点

```
MCU (SPI_HOST)                       存储芯片 (SPI_SLAVE)
┌──────────────┐                     ┌──────────────┐
│              │                     │              │
│     DIO ─────┼──────[330Ω]────┬────┼── SI          │
│              │                │    │              │
│              │                │    │              │
│              │                └────┼── SO          │
│              │                     │              │
│    SCLK ─────┼────────────────────┼── SCLK        │
│     CS# ─────┼────────────────────┼── CS#         │
└──────────────┘                     └──────────────┘
```

**布线规则**：
- DIO 走线长度 < 5cm（高频时考虑阻抗匹配）
- 避免与高速信号（USB/SDIO）平行走线
- CLK 线优先包地处理
- 330Ω 电阻两端预留测试点，方便调试

### 6.3 信号完整性验证

**测试项**：
1. **静态测试**：万用表测量 SI/SO 间阻值（应 > 1MΩ，确认内部未短接）
2. **上电测试**：示波器观察初始化阶段 DAT0/SO 电平，确认非读阶段为高阻
3. **动态测试**：
   - 发送 0xFF 命令时，SO 应被拉高（上拉电阻作用）
   - 发送读命令后，SO 应出现有效数据波形
4. **冲突测试**：故意在发送阶段读取 SO，观察 330Ω 电阻限流效果（电流应 < 10mA）

## 7. 软件配置规范

### 7.1 通用配置模板

```c
// 板级配置：单线半双工
#define TCFG_FLASH_DEV_FLASH_READ_WIDTH  1   // 固定 1bit，禁用多线模式

// SPI 平台数据结构
static const struct spi_platform_data flash_spi_pdata = {
    .port = {
        [0] = IO_PORTA_00,    // CS#  引脚
        [1] = IO_PORTA_01,    // DIO  引脚（复用，SDO/SDI 合并）
        [2] = IO_PORTA_01,    // 必须与 [1] 同值，标识 DIO 单线复用
        [3] = IO_PORTA_03,    // SCLK 引脚
    },
    .mode = SPI_MODE_UNIDIR_1BIT,   // 半双工，DIO 分时收发
    .clk  = 25000000,               // 25MHz，保守速率
};
```

### 7.2 驱动代码兼容性

**关键原则**：驱动代码写法不变，只改硬件配置。

```c
// 标准 SPI 读写函数，无需感知物理层是单线还是双线
void flash_read_id(uint8_t *id_buf) {
    spi_cs_low();                   // 拉低片选
    spi_write_byte(0x9F);           // 发读 ID 命令 → 控制器自动切 DIO 为输出
    id_buf[0] = spi_recv_byte();    // 收 Manufacturer ID → 控制器自动切 DIO 为输入
    id_buf[1] = spi_recv_byte();    // 收 Memory Type
    id_buf[2] = spi_recv_byte();    // 收 Memory Capacity
    spi_cs_high();                  // 拉高片选
}
```

**HAL 层要求**：
- SPI 控制器必须支持 `SPI_MODE_UNIDIR_1BIT` 或等效半双工模式
- 方向切换由硬件自动完成，软件无需干预
- 若 HAL 不支持，需手动切换 GPIO 方向（不推荐，时序风险高）

## 8. 风险与缓解措施

### 8.1 风险矩阵

| 风险项 | 概率 | 影响 | 缓解措施 | 残余风险 |
|--------|------|------|---------|---------|
| SO 非三态导致总线冲突 | 中 | 高（器件损坏） | 上电前用万用表确认 SI/SO 独立性 | 低 |
| 上拉电阻与 MCU 驱动冲突 | 中 | 中（逻辑错误） | 降低上拉阻值或改用开漏输出 | 低 |
| 方向切换时序错误 | 低 | 高（数据错乱） | 使用硬件自动切换的 HAL 模式 | 极低 |
| 信号完整性退化 | 低 | 中（误码率上升） | 控制走线长度 < 5cm，CLK 包地 | 低 |
| 禁用多线模式后速度不足 | 高 | 中（功能受限） | 前期评估带宽需求，预留双线焊盘 | 可接受 |

### 8.2 调试 checklist

- [ ] 确认芯片数据手册中 SO 类型为 "O" 或 "O/PP" 且支持 Hi-Z
- [ ] 测量 SI/SO 间阻值 > 1MΩ
- [ ] 确认 MCU SPI 控制器支持单线半双工模式
- [ ] 配置 `SPI_MODE_UNIDIR_1BIT` 并验证编译通过
- [ ] 上电后示波器观察 CS/CLK/DIO 波形正常
- [ ] 发送读 ID 命令，确认返回数据正确
- [ ] 进行 1000 次读写压力测试，无异常
- [ ] 高低温测试（-20°C ~ +60°C）通过

## 9. 总结

### 9.1 核心结论

**单线复用半双工模式是一种在引脚受限场景下经过验证的成熟方案**，其成立基于三个物理事实：

1. **存储芯片的 SPI 通信天然是半双工交替**（命令 → 地址 → 数据，单向流动）
2. **从机的 SO 引脚在非读阶段处于高阻态**，不会干扰主机发送
3. **一颗 330Ω 电阻**即可在方向切换瞬间提供短路保护

### 9.3 设计哲学

> "在资源受限的嵌入式设计中，**每省一个 GPIO 都是一次架构胜利**。单线 SPI 方案用一颗电阻换取一个引脚，是硬件工程师在约束条件下寻求最优解的典型体现。"

**关键提醒**：
- 单线方案**不是银弹**，它牺牲了速度和扩展性
- 设计时**预留双线焊盘**作为兼容选项，便于后期升级
- 量产前必须进行高低温、电压边界、长时间稳定性验证
