# 5 物理 IO 回弹按键接入按键事件框架 — 实现记录

## 1. 架构总览

```
硬件 GPIO                     SDK 键值               RDX 映射表               RDX 消息
──────────────────────────────────────────────────────────────────────────────────
PC2 (KEY1)  ──→ KEY_IO_NUM0 (65) ──→ key_table_io_num0_normal ──→ APP_MSG_REC_FR (快退)
PG7 (KEY2)  ──→ KEY_IO_NUM1 (66) ──→ key_table_io_num1_normal ──→ APP_MSG_REC_FF (快进)
PB2 (KEY3)  ──→ KEY_IO_NUM2 (67) ──→ key_table_io_num2_normal ──→ APP_MSG_VOL_UP (音量加)
PB4 (KEY4)  ──→ KEY_IO_NUM3 (68) ──→ key_table_io_num3_normal ──→ APP_MSG_RECORD_SWITCH (录音开关)
PG8 (KEY5)  ──→ KEY_IO_NUM4 (69) ──→ key_table_io_num4_normal ──→ APP_MSG_RECORD_SWITCH (录音开关)
```

事件流：

```
key_event_handler() → MSG_FROM_KEY → rdx_app_key_msg_handler()
  → rdx_app_earphone_key_remap():
      if KEY_IO_NUM0~4 → rdx_key_get_io_num_table(num_idx, scene) → 查表返回消息
  → return true  (中断分发, mode key table 收不到)
```

## 2. 核心原则

1. **零入侵 key_driver.h** — 复用 SDK 已有的 `KEY_IO_NUM0~4` (值 65-69)
2. **键值只标识物理来源** — 不携带功能语义
3. **RDX 统一管理映射** — `rdx_app_key_msg_handler` 返回 `true`, mode key table 被绕过
4. **配置单点集中** — `sdk_config.c:g_iokey_info[]` 配 GPIO, `rdx_key.c` 配功能

## 3. 改动清单

### 3.1 `sdk_config.c` — 5 键键值

```c
const struct iokey_info g_iokey_info[] = {
    { .key_value = KEY_IO_NUM0, .key_io = IO_PORTC_02 },  // KEY1
    { .key_value = KEY_IO_NUM1, .key_io = IO_PORTG_07 },  // KEY2
    { .key_value = KEY_IO_NUM2, .key_io = IO_PORTB_02 },  // KEY3
    { .key_value = KEY_IO_NUM3, .key_io = IO_PORTB_04 },  // KEY4
    { .key_value = KEY_IO_NUM4, .key_io = IO_PORTG_08 },  // KEY5
};
```

### 3.2 `iokey_config.c/h` — 删除 UUID 死代码

`key_uuid_table[]` 和 `uuid2keyValue()` 在整个 SDK 中零调用, 已删除。
`get_iokey_platform_data()` / `is_iokey_press_down()` / `get_iokey_power_io()` 仍然活跃。

### 3.3 `rdx_key.c` — 5 张 IO NUM 映射表

每键一张 `KEY_ACTION_MAX` 长度表, 格式与业务按键表对齐 (逐行 + 中文注释):

- **NUM0 (KEY1, PC2)**: `APP_MSG_REC_FR` @ LONG[1]
- **NUM1 (KEY2, PG7)**: `APP_MSG_REC_FF` @ LONG[1]
- **NUM2 (KEY3, PB2)**: `APP_MSG_VOL_UP` @ CLICK[0] + LONG[1]
- **NUM3 (KEY4, PB4)**: `APP_MSG_RECORD_SWITCH` @ LONG[1], `APP_MSG_LONG_PRESS_HOLDUP` @ UP[3]
- **NUM4 (KEY5, PG8)**: `APP_MSG_RECORD_SWITCH` @ LONG[1], `APP_MSG_LONG_PRESS_HOLDUP` @ UP[3]

KEY5 录音触发链: LONG → `APP_MSG_RECORD_SWITCH` → 置 flag=1, UP → `APP_MSG_LONG_PRESS_HOLDUP` → 启动录音。

查询函数:

```c
static u8 *g_num_normal_tables[] = {
    key_table_io_num0_normal,
    key_table_io_num1_normal,
    key_table_io_num2_normal,
    key_table_io_num3_normal,
    key_table_io_num4_normal,
};

u8 *rdx_key_get_io_num_table(int num_idx, int scene)
{
    if (num_idx < 0 || num_idx > 4) return NULL;
    switch (scene) {
    case 4:  // DUT  — 物理按键不干扰测试
    case 5:  // OTA  — 物理按键不干扰升级
        return NULL;
    default:
        return g_num_normal_tables[num_idx];
    }
}
```

### 3.4 `rdx_app.c` — 按键分发 + 场景判断

`rdx_app_earphone_key_remap()`:

```c
// IO NUM 键值分发
if (key->value >= KEY_IO_NUM0 && key->value <= KEY_IO_NUM4) {
    int num_idx = key->value - KEY_IO_NUM0;
    int scene = rdx_app_get_scene();
    pk_r = rdx_key_get_io_num_table(num_idx, scene);
    if (pk_r) *value = pk_r[index];
    return;
}
// KEY_POWER 原有 TWS L/R 逻辑继续走下面
```

`rdx_app_get_scene()`:

```c
int rdx_app_get_scene(void)
{
    RecordStatus* rp = rdx_record_get_status();
#if RDX_WIFI_ENABLE
    RdxWifiInfo* p = rdx_app_get_wifi_info();
#endif

    if (app_in_mode(APP_MODE_IDLE))  return 0;   // IDLE
    if (app_is_idle == TRUE)         return 0;
    if (get_ota_status())            return 5;   // OTA
    if (rdx_dut_mode)                return 4;   // DUT
#if RDX_WIFI_ENABLE
    if (p->onoff == TRANSFER_BY_WIFI_ON) return 3;   // WIFI
#endif
    if (rp->run == RECORD_STATE_START || rp->run == RECORD_STATE_RESUME)
                                     return 2;   // RECORDING
    return 1;   // NORMAL
}
```

## 4. 键值分配

| 按键 | IO   | SDK 键值          | 当前功能                 |
|------|------|-------------------|------------------------|
| KEY1 | PC2  | `KEY_IO_NUM0` (65) | 长按快退                |
| KEY2 | PG7  | `KEY_IO_NUM1` (66) | 长按快进                |
| KEY3 | PB2  | `KEY_IO_NUM2` (67) | 短按/长按 音量加        |
| KEY4 | PB4  | `KEY_IO_NUM3` (68) | 长按录音开关 + 抬起触发 |
| KEY5 | PG8  | `KEY_IO_NUM4` (69) | 长按录音开关 + 抬起触发 |

## 5. 注意事项

### 5.1 TCFG_IOKEY_ENABLE 前置条件

`app_config.h` 中 `BT_NORMAL` 模式 IOKEY=1; BQB/PER/FCC/DUT 等测试模式 IOKEY 被关闭——这是 SDK 原有行为。

### 5.2 `is_iokey_press_down()` / `get_iokey_power_io()`

只扫描 `KEY_POWER`, 不影响 5 键和拨码开关。

### 5.3 KEY3/KEY4 音量消息

`APP_MSG_VOL_UP` 当前仅 KEY3 使用。`rdx_app_msg_handler()` 暂无 VOL_UP/DOWN handler, 后续需补充。

### 5.4 按键扫描特性

JL key scan 驱动对持续按下只产生 LONG(750ms)/HOLD(900ms+)/UP, 不产生 CLICK。因此主要动作放在 LONG[1], CLICK[0] 留给多击场景。
