  第二层：1 秒后 rdx_app_bt_shutdown() 摧毁所有经典蓝牙能力

  // rdx_app.c:1112
  void rdx_app_bt_shutdown(void)
  {
      clr_device_in_page_list();                           // 清空 page 列表

      bt_cmd_prepare(USER_CTRL_POWER_OFF, 0, NULL);        // 关闭蓝牙协议栈！

      dac_power_off();                                     // 关 DAC
      // ...
  }

  JL 原生在第 583 行刚打开的 Page Scan，1 秒后就被 RDX 的 USER_CTRL_POWER_OFF 彻底关掉了。

  时间线
    │
    ├─ T0: BT_STATUS_INIT_OK
    │     ├─ JL 原生: lmp_hci_write_scan_enable(0x03)  → Page Scan ON  ✅
    │     └─ RDX:     sys_timeout_add(rdx_app_bt_shutdown, 1000)

    │     ├─ clr_device_in_page_list()
    │     ├─ bt_cmd_prepare(USER_CTRL_POWER_OFF, ...)  → BT 协议栈关闭 ❌
    │     └─ Page Scan 彻底消失
    │
    ├─ 系统后续尝试 APP_MSG_BT_OPEN_PAGE_SCAN
    │     └─ RDX prob handler → 空 stub → 什么都不做
    │
    └─ 结果：经典蓝牙永远不可被发现、不可被连接

  录音笔为什么这样设计

  录音笔走 BLE 连接手机 APP，不需要 Classic BT 可见。BT_STATUS_INIT_OK 后 1 秒关机是一种"快速进入低功耗 idle 等待 BLE 连接"的策略。

  对 MINI 语音键盘的影响

  这条链路解释了为什么即使 JL 原生在 BT_STATUS_INIT_OK 里打开了 Page Scan，也会被 RDX 在 1 秒后关掉。要让 Classic HID 配对工作，至少需要：

  1. 阻止 rdx_app_bt_shutdown() 被调度，或让它不执行 USER_CTRL_POWER_OFF
  2. 在合适的时机调用 rdx_app_bt_open()（或直接调 lmp_hci_write_scan_enable）
  3. 修复 rdx_app_earphone_state_set_page_scan_enable() 空 stub
