# 如何在原理图中区分 AD 按键与 IO 按键

> 本文档以 T2620-MMI-V00（JL7018F6 平台）原理图为例，系统讲解如何在电路原理图中识别和区分 AD（模拟量）按键与 IO（数字量）按键。

---

## 一、两种按键方案的基本概念

### 1. IO 按键（独立 GPIO 按键）

每个按键单独占用一个 GPIO 引脚，按键一端连接 GPIO，另一端连接 GND（或 VCC）。按下时 GPIO 电平翻转，MCU 直接检测高低电平变化。

```
MCU_GPIO1 ────┬──── 按键1 ──── GND
MCU_GPIO2 ────┼──── 按键2 ──── GND
MCU_GPIO3 ────┼──── 按键3 ──── GND
MCU_GPIO4 ────┼──── 按键4 ──── GND
MCU_GPIO5 ────┴──── 按键5 ──── GND
```

**特点：**
- 每个按键独占 1 个 GPIO 口
- N 个按键需要 N 个 GPIO 口
- 按键之间相互独立，无冲突
- 支持同时按下多个按键（组合键）
- 软件处理简单，直接读取 GPIO 电平

---

### 2. AD 按键（ADC 分压按键 / 电阻网络按键）

多个按键共用 **一个 ADC 引脚**，通过不同按键串联不同阻值的电阻形成分压网络。按下不同按键时，ADC 采集到的电压值不同，MCU 通过电压值判断哪个按键被按下。

```
           R1      R2      R3      R4      R5
VCC ──┬───[1K]───[2K]───[4.7K]──[10K]──[22K]───┐
      │    │      │       │       │       │     │
      │   按键1  按键2   按键3   按键4   按键5  GND
      │    │      │       │       │       │
      └────┴──────┴───────┴───────┴───────┴──── ADC_IN
```

**特点：**
- N 个按键只占用 1 个 ADC 口
- 极大节省 GPIO 资源
- 不能同时检测多个按键（同时按多个会导致电压混乱）
- 需要 ADC 采样和电压判断，软件稍复杂
- 电阻精度影响按键识别准确度

---

## 二、在原理图中区分的五大关键特征

### 特征1：看引脚数量 —— 最直观的判断方法

| 对比项 | IO 按键 | AD 按键 |
|--------|---------|---------|
| 按键数量 | N 个 | N 个 |
| **占用引脚数** | **N 个 GPIO** | **1 个 ADC** |
| 引脚类型 | GPIO/IO口 | ADC_IN |

**实例分析（T2620-MMI-V00）：**

从原理图第 501-505 行可以看到：

```
[1]KEY5
[1]KEY2 VDD_EN [2] GND
KEY1[1]
[1]KEY4
[1]KEY3
```

KEY1~KEY5 分别标注了 **5 个独立的信号名**，且各自连接到 JL7018F6 的不同引脚。如果是 AD 按键，5 个按键应该汇聚到 **同一个 ADC 引脚**，只会有 1 个信号名。

**结论：5 个独立引脚 = IO 按键。**

---

### 特征2：看电阻连接方式 —— 最关键的判断依据

#### IO 按键的电阻特征：

```
GPIO ────[0Ω/10Ω/100Ω]──── 按键 ──── GND
```

- 每个按键串联 **一个独立的电阻**
- 电阻阻值通常很小：**0Ω（跳线）、10Ω、100Ω**
- 电阻的作用：ESD 防护或线路限流，**不参与分压**
- 电阻直接连接在 GPIO 和按键之间，**彼此不串连**

#### AD 按键的电阻特征：

```
ADC ────┬────[R1]──── 按键1 ──── GND
        ├────[R2]──── 按键2 ──── GND
        ├────[R3]──── 按键3 ──── GND
        └────[R4]──── 按键4 ──── GND
```

- 所有按键的 **一端共同连接到同一个 ADC 引脚**
- 每个按键串联 **不同阻值的电阻**（如 1K、2.2K、4.7K、10K 等）
- 电阻的作用是**产生不同分压**，用于区分不同按键
- 电阻阻值差异明显，呈阶梯分布

**实例分析（T2620-MMI-V00）：**

原理图中 KEY 区域的电阻：

```
µç×è0201 0¦¸ ¡À5% 0.05W   ← R15 = 0Ω
µç×è0201 0¦¸ ¡À5% 0.05W   ← R16 = 0Ω
µç×è0201 0¦¸ ¡À5% 0.05W   ← R25 = 0Ω
```

所有按键串联的电阻都是 **0Ω（零欧姆跳线电阻）**，这显然不是为了分压。0Ω 电阻仅起到跳线/限流作用。

**结论：0Ω 小电阻且各自独立 = IO 按键。**

---

### 特征3：看网络连接拓扑 —— 结构上的本质区别

#### IO 按键的网络结构：

```
         GPIO1 ── KEY1 ── GND   ← 独立网络1
         GPIO2 ── KEY2 ── GND   ← 独立网络2
         GPIO3 ── KEY3 ── GND   ← 独立网络3
         GPIO4 ── KEY4 ── GND   ← 独立网络4
         GPIO5 ── KEY5 ── GND   ← 独立网络5
```

- 每个按键形成 **独立的信号网络**
- 各按键之间 **没有电气连接**
- 在 PCB 上各走各的线，互不交叉汇聚

#### AD 按键的网络结构：

```
              ┌─ KEY1 ── R1 ──┐
              ├─ KEY2 ── R2 ──┤
  ADC_IN ─────┼─ KEY3 ── R3 ──┼──── 可能还有上拉/下拉
              ├─ KEY4 ── R4 ──┤
              └─ KEY5 ── R5 ──┘
```

- 所有按键 **一端汇聚到同一个 ADC_IN 网络**
- 所有按键共享一个 ADC 输入节点
- 在 PCB 上所有按键的电阻端连接在一起

**实例分析（T2620-MMI-V00）：**

从原理图可以看到 KEY1~KEY5 的信号标注格式：

```
KEY1[1]      ← 标注 [1] 表示网络编号
[1]KEY2      ← 每个 KEY 有独立的网络标号
[1]KEY3
[1]KEY4
[1]KEY5
```

如果共享同一个 ADC 网络，所有按键会标注为同一个网络名（如 `KEY_ADC[1]`），而不是各自独立的 `KEY1`、`KEY2`...

**结论：独立网络名、无汇聚节点 = IO 按键。**

---

### 特征4：看上拉/下拉电阻配置

#### IO 按键：

通常需要在 GPIO 上配置 **内部上拉电阻**（或外部添加上拉电阻）：

```
VCC ────[10K 上拉]────┬──── GPIO
                      │
                     按键
                      │
                     GND
```

- 未按下时：GPIO 被上拉至高电平
- 按下时：GPIO 被拉低至 GND
- 上拉电阻可以集成在 MCU 内部，原理图上不一定画出

#### AD 按键：

通常需要一个 **稳定的参考电压**（VCC），形成完整的分压回路：

```
VCC ────┬─────────────────┐
        │                 │
       [R上拉]           ADC_IN
        │                 │
        ├──[R1]─KEY1─GND  │
        ├──[R2]─KEY2─GND  │
        ├──[R3]─KEY3─GND  │
        └──[R4]─KEY4─GND  │
```

- 必须有上拉电阻连接到 VCC
- 形成 "VCC → 上拉电阻 → ADC_IN → 按键电阻 → GND" 的完整分压链
- ADC_IN 节点的电压由串联电阻网络决定

**实例分析（T2620-MMI-V00）：**

KEY 区域没有看到外部上拉电阻。JL7018F6 的 GPIO 内部集成了上拉/下拉功能，IO 按键可以直接使用内部上拉。

如果是 AD 按键，通常需要外部上拉电阻和明显的分压电阻网络，在原理图上会清晰标注。

**结论：无上拉/分压电阻网络 = IO 按键。**

---

### 特征5：看 BOM 清单中的电阻参数

#### IO 按键的 BOM 特征：

| 位号 | 阻值 | 精度 | 功能 |
|------|------|------|------|
| R15 | 0Ω | ±5% | 跳线/限流 |
| R16 | 0Ω | ±5% | 跳线/限流 |
| R25 | 0Ω | ±5% | 跳线/限流 |

- 电阻阻值小（0Ω、10Ω、100Ω）
- 所有按键的电阻**阻值相同**
- 电阻精度要求低

#### AD 按键的 BOM 特征：

| 位号 | 阻值 | 精度 | 功能 |
|------|------|------|------|
| R1 | 1KΩ | ±1% | 分压 |
| R2 | 2.2KΩ | ±1% | 分压 |
| R3 | 4.7KΩ | ±1% | 分压 |
| R4 | 10KΩ | ±1% | 分压 |
| R5 | 22KΩ | ±1% | 分压 |

- 电阻阻值差异大，呈阶梯分布
- 每个按键的电阻**阻值不同**
- 电阻精度要求高（±1% 或更高），确保分压准确

**实例分析（T2620-MMI-V00）：**

原理图中 KEY 相关电阻标注：

```
µç×è0201 0¦¸ ¡À5% 0.05W
```

- 阻值：0Ω
- 精度：±5%（普通精度）
- 功率：0.05W（小信号）

这明显是跳线电阻，不是分压电阻。

**结论：BOM 中电阻阻值相同且为 0Ω = IO 按键。**

## 四、T2620-MMI-V00 实例总结

| 判断特征 | 实际观察结果 | 判断 |
|----------|-------------|------|
| 引脚数量 | KEY1~KEY5 各占用独立引脚，共 5 个 GPIO | IO 按键 |
| 电阻阻值 | R15/R16/R25 = 0Ω（跳线电阻） | IO 按键 |
| 电阻数量 | 每个按键 1 个独立电阻，互不相连 | IO 按键 |
| 网络拓扑 | 5 个独立的信号网络，无汇聚节点 | IO 按键 |
| 上拉/分压 | 无分压电阻网络 | IO 按键 |
| BOM 精度 | ±5% 普通精度 | IO 按键 |

### 最终结论：

**T2620-MMI-V00 的 KEY1~KEY5 是 5 个独立的 IO 按键**，分别连接到 JL7018F6 的 5 个独立 GPIO 口。每个按键串联一个 0Ω 跳线电阻（ESD/限流用），通过检测 GPIO 电平高低来判断按键是否按下。

## 五、常见误区与注意事项

### 误区1：看到有电阻就认为是 AD 按键

**正解：** IO 按键也可能串联电阻（如 0Ω 跳线、100Ω 限流电阻），关键看电阻的**阻值和连接方式**。

### 误区2：认为独立 IO 按键不需要电阻

**正解：** 很多设计会在按键回路中串联 0Ω 电阻或 ESD 保护器件，这是为了生产测试和静电防护，与 AD 分压无关。

### 误区3：以为所有产品都用 AD 按键省 GPIO

**正解：** 对于 GPIO 资源充裕的芯片（如 JL7018F6），独立 IO 按键更简单可靠。AD 按键通常用于引脚极其紧张的小型 MCU。

### 误区4：只看电阻不看网络名

**正解：** 原理图中的网络名（Net Label）是判断信号走向的关键。AD 按键所有按键会共享同一个 ADC 网络名。

## 六、附录：两种方案的典型应用场景

| 场景 | 推荐方案 | 原因 |
|------|----------|------|
| GPIO 充裕、功能复杂（如蓝牙耳机） | IO 按键 | 支持组合键、长按短按、软件简单 |
| GPIO 紧张、引脚极少的 MCU | AD 按键 | 节省引脚资源 |
| 需要同时按多个键（如游戏手柄） | IO 按键 | AD 按键无法同时检测 |
| 成本敏感、按键多（如遥控器） | AD 按键 | 减少引脚数量，降低封装成本 |
| 抗干扰要求高 | IO 按键 | 数字量抗干扰优于模拟量 |
